    -> Accessing Strings 

    -> Indexing 

        -> String indexing in Python allows access to "individual characters within a string" 
            using their "position", known as the "index".
        
        -> Python uses zero-based indexing, meaning the first character is at index 0, 
            the second at index 1, and so on.

        -> Positive Indexing:
            Starts from the beginning of the string.
            The index of the first character is 0.
        
        -> Negative Indexing:
            Starts from the end of the string.
            The index of the last character is -1.

        -> text = "python"

            0  1  2  3  4  5 [ Positive Indexing ]
            p  y  t  h  o  n 
           -6  -5 -4 -3 -2 -1  [ Negative Indexing ]

        -> syntax 

            string[index]

            -> NOTE : Accessing invalid index will raise error (IndexError)
    
    -> Slicing

        -> String slicing in Python is a method for extracting a portion (substring) 
        from a larger string. It utilizes the slice operator [] with specific 
        indices to define the desired segment.

        -> syntax 

            string[start:end:step]

                -> start (optional): The index where the slice begins (inclusive). 
                    If omitted, it defaults to 0 (the beginning of the string).
                -> end (optional): The index where the slice ends (exclusive). 
                    The character at this index is not included. 
                    If omitted, it defaults to the length of the string (the end of the string).
                -> step (optional): The increment between each index during slicing. 
                    If omitted, it defaults to 1. 
                    A negative step value can be used to slice in reverse.
            
            -> Earlier we used range()

                range(start,end,step)

            simpler manageable sub tasks for promoting code modularity. 

        -> Once defined a function, it can be called multiple times, so that 
            we can achieve code reusability.

        -> Functions are defined using "def" keyword, followed by function name,
        parenthesis(which may contain parameters), and a colon:
        the code within the function is indented 

        -> syntax

            def function_name():
                statements

            def function_name(parameters):
                statements

        -> NOTE: calling function is must

            function_name()

    -> Python functions can be defined to accept "parameters" 
        which can act as placeholders for values that will be 
        passed into function when called. 
    
    -> Values passed to the function we call them as "arguments"

    -> We use parameters to make functions dynamic in nature 

    -> Based On arguments passed to functions, we got five types 

        -> Positional Arguments 

        -> Keywords Arguments 

        -> Default Arguments 

        -> Arbitrary Positional Arguments

        -> Arbitrary Keywords Arguments

    -> Positional Arguments

        -> In Python, positional arguments are values passed to a function during 
        its call, where the assignment of these values to the function's parameters 
        is determined solely by their order (position) in the argument list. 
        
        -> The first argument in the function call is assigned to the first parameter 
        in the function definition, the second to the second, and so on. 

        -> Order Matters: If the order is incorrect, the function will receive the wrong values 
            for its intended parameters, potentially leading to errors or unexpected behavior.

    
    -> Keywords Arguments

        -> In Python, keyword arguments are a way to pass values to a function by explicitly 
        "naming the parameters" they correspond to, rather than relying on their position 
            in the argument list. 
        
        -> This offers several advantages, primarily in terms of code readability and flexibility.
    
    -> Default Arguments 

        -> In Python, default arguments allow you to define a function parameter 
            with a "pre-assigned" value. 
        
        -> If a caller does not provide an argument for that parameter during a function call, 
            the default value is automatically used. 
            If an argument is provided, it overrides the default.

        -> Placement Requirement: Default arguments must always be placed after 
            any non-default (positional) arguments in the function definition. 
            Attempting to place a non-default argument after a 
                default argument will result in a SyntaxError.

    -> Arbitrary Positional Arguments

        -> In Python, arbitrary positional arguments allow a function to accept 
            a variable/unknown number of non-keyword arguments. 
            This is achieved by placing an asterisk (*) before a parameter name 
                in the function definition. 
            
        -> This parameter will then collect all the extra positional arguments passed 
            to the function into a single tuple. 

    
    -> Arbitrary Keywords Arguments

        -> In Python, arbitrary keyword arguments allow a function to accept 
            a variable/unknown number of keyword arguments. 
            This is achieved by placing an asterisk (**) before a parameter name 
                in the function definition. 

        -> This parameter will be collected into a dictionary, where the names will be keys and 
            values will be dictionary values

    -> Exception Handling 

        -> Exception : It's an "Error", which might occur during the program execution 

        -> By default when python encounters an Error, it abruptly "STOPS" the program execution

    -> To change the above default behavior, and handle the Exceptions 
            by yourself, we need following

            -> try: used to keep the code, that will cause error 

            -> except: used to keep the code that should run, when errors occur 

                -> Try statement must have at least one except or finally clause

            -> else: used to keep the code that short run, if No Exception was raised in try block

            -> finally: used to keep the code that should run, whether an exception raised or not 
                        DB / Files -> closing connections will be in done in finally block 

    -> Earlier we worked with In built Exceptions provided as part of 
        python standard library (ZeroDivisionError, IndexError, ValueError etc)

        -> https://docs.python.org/3/library/exceptions.html 

    -> But sometimes you need your "own custom errors" that describes a situation 
        Specific to your program 
    
    -> You create a custom exception by "defining a class" that inherits 
        from Pythons built-in "Exception" class. 
        This concept is connected to inheritance.

        -> All user-defined exceptions should also be derived from Exception class.

        -> syntax

            class MyCustomError(Exception):
                pass
